Backbone.js

Небольшая библиотека кирпичиков для структурированной разработки

  • Models — данные которые можно создавать, удалять, изменять, валидировать и синхронизировать с сервером
  • Collections — набор моделей с удобным API
  • Views — элементы интерфейса отображающие состояние модели
  • Events — примесь, позволяет слушать и вызывать события
  • Router — роутинг на стороне клиента

Больше конкретики

Backbone следит за тем, чтобы ваши данные на всех трех уровнях были синхронизованны

Нажатие на ссылку → Роутер → Сервер → Коллекция\модель → Представление →
 Клиент взаимодействует с UI → Модель → Cервер

Backbone-relational

В моделях бэкбона нет связей

Вкусности:
Один-к-одному, один-ко-многим и многие-ко-многим отношения в моделях, ленивая загрузка и автоматическая конвертация

			 House = Backbone.RelationalModel.extend({
			  relations: [
			    {
			      type: Backbone.HasMany,
			      key: 'occupants',
			      relatedModel: 'Person',
			      includeInJSON: Backbone.Model.prototype.idAttribute,
			      collectionType: 'PersonCollection',
			      reverseRelation: { key: 'livesIn' }
			    }
			  ]
			});

Пример

			var persons = PersonCollection();
			presons.fetch({someFilter: 1}); // 2, 10
			var house = House({id: 5});
			house.fetch(); // occupants [2,10,8]
			// С сервера подтянулся только житель с id=8
			house.get('occupants').get(10); // Модель

Стандартные действия фреймворк должен делать сам

И давать хук по которому разработчик может почистить за собой

Модульность

Небольшие, инкапсулированные модули, сфокусированные на выполнении определенной задачи

Есть RequireJS, но иногда хочется большего

Необходим механизм инициализации и остановки модулей

Коммуникации

Модули должны взаимодействовать друг с другом

  • Event Aggregator
  • Publish/Subscribe
  • Mediator
  • Commands

Надстройки

  • LayoutManager — управление макетами и шаблонами
  • Chaplin — легковесная и гибкая структура для вашего приложения на основе популярных паттернов
  • Thorax — решающий за вас, проверенный в бою фреймворк для создания больших веб приложений
  • Marionette.js — архитектура и коллекции паттернов, делает жизнь разработчика проще

LayoutManager

Когда не нужны средства для сложной архитектуры,
модули и все эти абстракции

  • Полный жизненный цикл шаблонов
  • Структурированные макеты
  • Очистка представлений
  • API для вложенных View
  • Максимально прост в интеграции

Chaplin

  • Удобное наследование — автоматическое слияние декларативно-описанных событий
  • Управление памятью — следит за корректным уничтожением объектов
  • Модульность — заставляет использовать RequireJS, нет высокоуровневых абстракций, только контроллеры, инициализаторы присутствуют
  • Коммуникации — Dispatcher, EventBroker реализующий Pub/Sub, высокоуровневый Mediator — общая шина
  • Layout, умные View и CollectionView – необходимая база

Thorax

  • Handlebars – шаблонный помощник на каждый случай
  • Свои модели и коллекции
  • Удобный и гибкий базовый View, Layouts
  • Интеграция с jQuery & Zepto – по элементу можно получить модель, коллекцию или объект представления
  • Наследуемые события
  • Form Handling — просто превращаем формы в модели и обратно, хук для валидации

Marionette.js

Архитектура для приложения, коллекция паттернов, нет привязки к шаблонному движку, не обязательно использовать все модули

  • Views — ItemView, CollectionView, CompositeView, Layout
  • View Management — Region, RegionManagement
  • Infrastructure — Application, Module, Controller.
  • Object-Messaging — EventAggregator, RequestResponse, Commands
